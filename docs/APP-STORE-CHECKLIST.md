# App Store 发布检查清单

## 📋 发布前检查清单

### 一、资质准备 ✅/❌

- [ ] Apple 开发者账号（$99/年）
- [ ] 软件著作权证书

- [ ] Bundle ID 配置
- [ ] 应用图标（所有尺寸）
- [ ] 启动图（所有尺寸）
- [ ] 项目版本号更新

### 三、App Store Connect ✅/❌

- [ ] 创建应用记录
- [ ] 应用名称（确保不重复）
- [ ] 应用描述（4500字符内）
- [ ] 关键词（100字符内）
- [ ] 应用截图（各设备至少3张）
- [ ] 宣传图 1024x1024
- [ ] 隐私政策 URL
- [ ] 技术支持 URL
- [ ] 年龄分级问卷
- [ ] 导出合规信息

### 四、权限声明 ✅/❌

- [ ] 相机权限说明文字
- [ ] 相册权限说明文字
- [ ] 数据收集声明
- [ ] 第三方SDK声明

### 五、测试检查 ✅/❌

- [ ] 真机测试通过
- [ ] 所有功能可用
- [ ] 无崩溃和严重Bug
- [ ] 网络异常处理
- [ ] 弱网测试
- [ ] 不同机型适配

## ⚠️ 常见审核被拒原因

1. **功能不完整** - 确保所有功能可用
2. **崩溃或Bug** - 充分测试后再提交
3. **元数据问题** - 截图与实际不符
4. **隐私政策缺失** - 必须提供有效链接
5. **权限说明不清** - 需明确说明权限用途
6. **设计不规范** - 遵循 iOS 设计准则
7. **测试账号** - 如有登录需提供测试账号
8. **内购配置** - 内购项目需正确配置

## 🚀 发布步骤

### 1. HBuilderX 打包
```
1. 打开 HBuilderX
2. 选择项目 -> 发行 -> 原生App-云打包
3. 选择 iOS，上传证书和描述文件
4. 等待打包完成，下载 .ipa 文件
```

### 2. 上传到 App Store Connect
```
1. 使用 Transporter 或 Application Loader 上传 .ipa
2. 等待处理完成（通常 15-30 分钟）
3. 在 App Store Connect 选择构建版本
```

### 3. 提交审核
```
1. 填写"此版本的新增内容"
2. 填写联系方式
3. 点击"提交以供审核"
4. 等待审核（通常 1-3 天）
```

### 4. 上架
```
审核通过后，选择"手动发布"或"自动发布"
应用将在 24 小时内上架
```
